1.0.0.0.4 Training Exercise
  • Use tabula to extract Table 1 from this paper
  • Reformat the table using the guidelines in the journal article on effective data management
  • Preferable to use R to reformat the table
  • Use Morpho to write metadata for the data table, drawing on information in the published paper it came from, BUT change the “creator” (or author) to your own name
  • Save and export package locally, not to network
  • After you export the package, find the metadata file on the local machine. It will be in the morpho export folder, in the metadata sub folder, and will have a file name that looks like datateam2.3 with no extension. Change this filename to something more clear with an xml extension, like “science_metadata.xml”
  • Transfer the file to the datateam server using the cyberduck client
  • A note on paths: on the servers, paths to files in your folder always start with /home/yourusername/....
  • Submit the data to test.arcticdata.io using the web form (UI)
  • Submit the data to test.arcticdata.io using arcticdatautils
  • Compare your submission against the checklist and update it until it meets all the checks by:
  • Updating data files as necessary,
  • Manipulating the EML using the EML package,
  • and then updating the submission on the test site using arcticdatautils.
  • Finally, upload your script to the GitHub Enterprise repository by following these instructions
